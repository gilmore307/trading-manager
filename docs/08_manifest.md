# Manifest Contract

## Purpose

This file will define the system-level manifest contract used as run evidence across repositories.

A manifest records what ran, which inputs were used, which outputs were produced, what code/config was involved, and whether the output is usable.

## Initial Contract Boundary

This file should define:

- manifest identity and versioning;
- required run evidence;
- input artifact references;
- output artifact references;
- producer repository;
- workflow identifier;
- code/config evidence;
- validation evidence;
- compatibility expectations;
- relationship to ready signals.

## Open Gap

The exact manifest schema is not yet defined.
